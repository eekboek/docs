Systeemeisen

Elk modern GNU/Linux systeem is in staat om zonder problemen EekBoek te installeren en te draaien. Heeft u een recent Fedora, Debian of Ubuntu systeem, dan is EekBoek zelfs als standaardpakket beschikbaar.

Controleer eerst of de versie van Perl toereikend is. Open daartoe een terminal window en geef de opdracht:

perl --version
De uitvoer begint met de melding
This is perl, v5.8.8 built for ...
De Perl versie moet tenminste 5.8.6 zijn.

Installeren met YUM

Installatie met het installatieprogramma yum is de meest eenvoudige methode. Dit programma wordt op veel moderne Linux systemen standaard geleverd, en is anders vaak toe te voegen. Raadpleeg daartoe de systeemdocumentatie.

Het grootste voordeel van installatie met yum is dat dit programma zelf onthoudt waar de te installeren software vandaan moet worden gehaald, en automatisch de meest recente versie installeert.

De volgende stappen dienen te worden uitgevoerd als super-user root.

  1. Installeer de EekBoek OpenPGP key, waarmee de bestanden kunnen worden gecontroleerd op echtheid en correctheid:

    rpm --import [% site.gpgkey %]

  2. Open met de browser de yum repository [% site.yum %], zoek de laatste versie van Eekboek-yum, klik erop, en download de betreffende noarch.rpm kit.

  3. Verifieer de gedownloade kit:

    rpm -K EekBoek-yum-versie.noarch.rpm

    De versie is iets als 1.04-3. De uitvoer ziet er ongeveer zo uit:

    EekBoek-yum-1.04-3.noarch.rpm: (sha1) dsa sha1 md5 gpg OK

    Let er op dat in de uitvoer met name de tekst “gpg OK” voorkomt als teken dat de verificatie van de herkomst is geslaagd.

  4. Installeer de eekboek repository kit:

    rpm -ivh EekBoek-yum-versie.noarch.rpm

  5. Installeer EekBoek.

    Wilt u EekBoek gaan gebruiken met een SQLite database:

    yum install EekBoek

    Wilt u EekBoek gaan gebruiken met een PostgreSQL database:

    yum install EekBoek EekBoek-db-postgresql

    Als U de PostgreSQL ondersteuning installeert kunt nog steeds ook SQLite gebruiken. De keuze welke database wordt gebruikt wordt namelijk in een van de configuratiebestanden vastgelegd.

    Standaard wordt EekBoek geïnstalleerd inclusief de grafische shell. Mocht dit niet gewenst zijn dan kunt u in plaats van EekBoek het package EekBoek-cli te installeren.

    Het is mogelijk dat yum enkele aanvullende pakketten wil installeren.

Wilt u in de toekomst een nieuwe versie van EekBoek installeren, dan kan dit eenvoudig met de opdracht:

yum upgrade EekBoek-db-database

Hiermee wordt dan steeds de meeste recente versie geïnstalleerd.

Installeren met RPM

Als het systeem geen yum heeft, maar wel rpm, dan dient u steeds expliciet op te geven waar de te installeren software vandaan moet worden gehaald, alsmede de versie van de te installeren software.

Er zijn vijf EekBoek packages:

EekBoek-cli Dit is het basispackage, inclusief ondersteuning voor SQLite. Deze heeft u altijd nodig.
EekBoek-gui Dit is de grafische shell.
EekBoek-db-postgresql Dit package heeft u nodig als u EekBoek met de PostgreSQL database wilt gebruiken.
EekBoek-contrib Dit package bevat door de EekBoek gebruikers zelf ontwikkelde hulpmiddelen en bestanden.
EekBoek Dit is een pseudo-package waarmee u in één keer de packages cli, gui en contribkunt installeren.

De volgende stappen dienen uitgevoerd te worden als super-user root.

  1. Installeer de EekBoek OpenPGP key, waarmee de bestanden kunnen worden gecontroleerd op echtheid en correctheid:
    rpm --import [% site.gpgkey %]

  2. Open met de browser de yum repository [% site.yum %], zoek de laatste versie van Eekboek, klik erop, en download de betreffende noarch.rpm packages.

  3. Verifieer de kits:
    rpm -K EekBoek*-versie.noarch.rpm
    De uitvoer ziet er ongeveer zo uit:
    EekBoek-2.00-1.noarch.rpm: (sha1) dsa sha1 md5 gpg OK EekBoek-db-postgresql-2.00-1.noarch.rpm: (sha1) dsa sha1 md5 gpg OK EekBoek-cli-2.00-1.noarch.rpm: (sha1) dsa sha1 md5 gpg OK EekBoek-gui-2.00-1.noarch.rpm: (sha1) dsa sha1 md5 gpg OK EekBoek-contrib-2.00-1.noarch.rpm: (sha1) dsa sha1 md5 gpg OK

    Let er op dat in de uitvoer met name de tekst “gpg OK” voorkomt als teken dat de verificatie van de herkomst is geslaagd.

  4. Installeer de gewenste packages.

    rpm -ivh package-versie.noarch.rpm

    Het is mogelijk dat rpm aangeeft dat nog enkele aanvullende pakketten nodig zijn. Deze dient u dan eerst te installeren en vervolgens de installatie van EekBoek opnieuw uit te voeren.

Wilt u in de toekomst een nieuwe versie van EekBoek installeren, dan dienen de stappen 2 t/m 4 opnieuw te worden uitgevoerd, met uiteraard het nieuwe versienummer.

Overweeg ook om de optionele modules te installeren.

Installeren met APT-GET

Er zijn twee EekBoek packages:

eekboek-cli Dit is het basispackage, inclusief ondersteuning voor SQLite. Deze heeft u altijd nodig.
eekboek-gui Dit is de grafische shell.
eekboek-db-postgresql Dit package heeft u nodig als u EekBoek met de PostgreSQL database wilt gebruiken.
eekboek-contrib Dit package bevat door de EekBoek gebruikers zelf ontwikkelde hulpmiddelen en bestanden.
eekboek Dit is een pseudo-package waarmee u in één keer de packages cli, gui en contribkunt installeren.

Eekboek wordt standaard meegeleverd met Debian unstable. De verwachting is dat het vanaf ongeveer oktober 2008 zal worden meegeleverd met Debian testing en met afgeleide systemen zoals Ubuntu. Is dat bij u het geval dan kunt u direct doorgaan met punt 4.

Voor het installeren dient u als super-user root de volgende stappen uit te voeren.

  1. Installeer de EekBoek OpenPGP key, waarmee de bestanden kunnen worden gecontroleerd op echtheid en correctheid:

    wget -O - [% site.gpgkey %] | apt-key add -

    U hoeft dit uiteraard maar eenmalig te doen.

  2. Voeg de Debian repository toe aan de lijst van repositories. U kunt dit doen door de volgende opdracht uit te voeren:

    echo 'deb http://www.eekboek.nl/repo/debian/ ./' > /etc/apt/sources.list.d/eekboek.list

    U hoeft dit uiteraard maar eenmalig te doen.

  3. Update de repository index:

    apt-get update

  4. Installeer de gewenste packages.

    apt-get install eekboek

    Als U de PostgreSQL ondersteuning installeert kunt nog steeds ook SQLite gebruiken. De keuze welke database wordt gebruikt wordt namelijk in een van de configuratiebestanden vastgelegd.

    Het is mogelijk dat apt-get adviseert nog enkele aanvullende pakketten toe te voegen. Het verdient aanbeveling dit advies op te volgen.

Overweeg ook om de optionele modules te installeren.

Optionele modules voor meer gemak en functionaliteit

De volgende modules zijn niet strikt nodig, maar bieden wel meer gebruiksgemak en functionaliteit.

Term::ReadLine::Gnu

Als u deze module installeert kunt u in de EekBoek shell gebruik maken van handige completion, waarbij automatisch dagboeken, rekeningnummers en relatiecodes worden opgezocht en aangevuld.

Optionele modules kunnen doorgaans als een standaard package via het normale installatiemechanisme worden geïnstalleerd.
