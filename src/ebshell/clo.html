Command line opties

-f cfg
--config=cfg		 Specificeert een extra configuratiebestand. Zie Configuratiebestanden.
-X De standaard-configuratiebestanden worden niet gelezen. Dit is uiteraard alleen zinvol in combinatie met --config. Zie Configuratiebestanden.
-P
--printcfg		 De rest van de opdrachtregel wordt geïnterpreteerd als namen van configuratie-variabelen en de bijbehorenden waardes worden uitgeprint. Zie Configuratiebestanden.
Deze optie moet als eerste op de commandoregel voorkomen.
-D var=waarde
--define var=waarde		 Definieer configuratievariabelen. ‘var’ is van de vorm ‘sectie:naam’
-c
--command		 De rest van de opdrachtregel wordt als een ebshell opdracht geïnterpreteerd.
-e
--echo		 In batch mode worden alle ingelezen commando's ook weggeschreven (ge-echo-t) naar de standaard uitvoer.
--dataset=name Deze optie komt te vervallen. Gebruik in plaats hiervan --config.
De opgegeven database wordt gebruikt in plaats van de standaard database. Zie Configuratiebestanden, sectie Database.
--boekjaar=code Selecteer een boekjaar uit een administratie die meerdere boekjaren omvat.
--init Initialiseer een nieuwe administratie.
Dit is gelijk aan de opties --import --dir ., maar gemakkelijker.
Zie de EekBoek opdracht import voor details.
--import import-parameters Importeer een administratie.
Zie de EekBoek opdracht import voor details.
Deze opdracht impliceert --createdb.
--export export-parameters Exporteer een administratie.
Zie de EekBoek opdracht export voor details.
--createdb Waarschuwing: alleen voor experts.
Creëer een lege database. Indien de database al bestaat wordt deze geheel leeggemaakt, anders wordt een nieuwe, lege database aangemaakt.
Deze opdracht is alleen zinvol in combinatie met de opdracht --schema, zie hieronder.
--schema=name Waarschuwing: alleen voor experts.
Initialiseer de administratie met het opgegeven schema. Dit kan uitsluitend indien de te gebruiken database nog helemaal leeg is.
Als de opgegeven schemanaam alleen letters, cijfers en onderstrepingen bevat wordt het schema gezocht in de volgende plaatsen:
./naam.dat
schema/naam.dat
EBLIB/schema/naam.dat
In het laatste geval staat EBLIB voor de locatie waar EekBoek is geïnstalleerd.
Als de opgegeven schemanaam andere tekens bevat wordt het als een bestandsnaam beschouwd en wordt het schema vanuit dat bestand geladen.
